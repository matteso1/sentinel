Distributed log streaming engine.

A high-throughput message queue built from scratch in Go. Custom LSM-tree storage engine, Raft-inspired consensus, gRPC streaming API. Kafka-style semantics without the JVM complexity.

Architecture

Producers send records to topic partitions. The broker persists data through an LSM-tree storage engine and replicates via Raft consensus. Consumers read from partitions at their own pace. 

graph LR
    P[Producer] --> B[Broker]
    B --> C[Consumer]
    B --> S[LSM Storage]
    S --> WAL[Write-Ahead Log]
    S --> MT[MemTable]
    MT --> SST[SSTables]

Write Path

Records are first written to a write-ahead log for durability, then inserted into an in-memory skip list. Background compaction flushes data to immutable SSTables on disk. 

sequenceDiagram
    participant Client
    participant Broker
    participant WAL
    participant MemTable
    participant SSTable

    Client->>Broker: Produce(topic, records)
    Broker->>WAL: Append(entry)
    WAL-->>Broker: Success
    Broker->>MemTable: Put(key, value)
    Note over MemTable: Skip list O(log n)
    MemTable-->>Broker: Offset
    Broker-->>Client: ProduceResponse

    Note over MemTable,SSTable: Background flush
    MemTable->>SSTable: Flush when full

Raft Consensus

Leader election uses randomized timeouts to prevent split-brain. Log entries replicate via AppendEntries RPC. Commits advance after majority acknowledgment. 

stateDiagram-v2
    [*] --> Follower
    Follower --> Candidate: Election timeout
    Candidate --> Leader: Majority votes
    Candidate --> Follower: Higher term discovered
    Leader --> Follower: Higher term discovered
    Leader --> Leader: Heartbeats

Components

internal/storage/skiplist.go O(log n) concurrent skip list with RWMutex. XorShift64 PRNG for level generation.
internal/storage/wal.go Write-ahead log with CRC32 checksums. Configurable sync modes. Crash recovery.
internal/storage/sstable.go Block-based sorted string tables. Binary search index. Key range tracking.
internal/storage/lsm.go LSM coordinator. Memtable rotation, background flush, multi-level storage.
internal/broker/broker.go Topic/partition management. Produce and consume operations. Persistence layer.
internal/broker/consumer_group.go Round-robin partition assignment. Offset tracking. Heartbeat and rebalancing.
internal/raft/node.go Raft state machine. Follower/Candidate/Leader states. Election timer.
internal/raft/replication.go AppendEntries RPC. Conflict detection. Commit index advancement.
internal/server/grpc_server.go Protobuf service. Produce, Consume (streaming), FetchMetadata RPCs.
internal/metrics/prometheus.go Counters, gauges, latency tracking. /metrics endpoint for Prometheus.

Quick Start

terminal
# clone
git clone https://github.com/matteso1/sentinel.git
cd sentinel

# start server
go run ./cmd/sentinel-server --port 9092 --data ./data
# Sentinel server listening on port 9092

# produce
go run ./cmd/sentinel-cli produce -topic orders -message "order:123" -count 100
# ✓ Produced 100 record(s) to orders (offset: 0)

# consume
go run ./cmd/sentinel-cli consume -topic orders -from-beginning
# offset=0 | key=(null) | value=order:123-0
# offset=1 | key=(null) | value=order:123-1
# ...
