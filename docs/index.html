The message queue you can actually understand.

Sentinel is a distributed log streaming engine built from scratch in Go. It delivers Kafka-style semantics—topics, partitions, consumer groups—without the operational complexity of running a JVM cluster. Perfect for developers who need reliable message passing without the infrastructure overhead.

The Problem

Message queues shouldn't require a PhD to operate.

Kafka is powerful but comes with significant operational overhead. ZooKeeper coordination, JVM tuning, broker configuration, partition rebalancing— it's a lot to manage when you just want to pass messages between services.

Architecture

Built on proven distributed systems primitives.

Sentinel combines a custom LSM-tree storage engine with Raft-inspired consensus to deliver durability and fault tolerance. Every design decision prioritizes simplicity without sacrificing correctness.

Write Path

When a producer sends a message, it's immediately written to a write-ahead log (WAL) for durability, then inserted into an in-memory skip list for fast reads. Background compaction flushes data to immutable SSTables on disk.

  • Skip List Memtable — O(log n) concurrent inserts with RWMutex protection
  • Write-Ahead Log — CRC32 checksums, configurable fsync modes
  • SSTable Format — Block-based layout with binary search index
  • Background Compaction — Automatic merging to reclaim space

Consensus

Leader election uses randomized timeouts to prevent split-brain scenarios. Log entries are replicated via AppendEntries RPC, with commits advancing only after majority acknowledgment.

Features

Everything you need, nothing you don't.

Custom Storage Engine

Hand-written LSM-tree with skip list, SSTables, and write-ahead logging. CRC32 checksums ensure data integrity. Crash recovery rebuilds state from the WAL.

gRPC Streaming API

Protobuf-defined service with Produce, Consume (server streaming), and FetchMetadata RPCs. Efficient binary serialization with strong typing.

Topics and Partitions

Kafka-style data organization. Messages are grouped by topic and distributed across partitions. Hash or round-robin partitioning strategies.

Consumer Groups

Automatic partition assignment across group members. Round-robin rebalancing when consumers join or leave. Offset tracking for exactly-once semantics.

Raft Consensus

Leader election with randomized timeouts. AppendEntries RPC for log replication. Commit index advances after majority acknowledgment.

Prometheus Metrics

Built-in /metrics endpoint exposing throughput, latency, error rates, and partition lag. Ready for Grafana dashboards.

Quick Start

From zero to streaming in under a minute.

terminal 
# Clone and build
git clone https://github.com/matteso1/sentinel.git
cd sentinel

# Start the server
go run ./cmd/sentinel-server --port 9092 --data ./data
# Sentinel server listening on port 9092

# In another terminal: produce messages
go run ./cmd/sentinel-cli produce \
    -topic orders \
    -message "order:12345" \
    -count 100
# ✓ Produced 100 record(s) to orders (offset: 0)

# Consume messages
go run ./cmd/sentinel-cli consume \
    -topic orders \
    -from-beginning
# offset=0 | key=(null) | value=order:12345-0
# offset=1 | key=(null) | value=order:12345-1
# ...
Use Cases

Built for real-world message passing.

Event Sourcing

Store all state changes as an append-only log. Replay events to rebuild application state. Perfect for audit trails and debugging.

Microservice Communication

Decouple services with asynchronous messaging. Producers and consumers evolve independently. No direct RPC dependencies.

Real-time Data Pipelines

Ingest clickstreams, IoT telemetry, or application logs. Buffer data for downstream analytics. Handle traffic spikes gracefully.

Task Queues

Distribute work across worker processes. Consumer groups ensure each task is processed once. Retry failed jobs by seeking to earlier offsets.
